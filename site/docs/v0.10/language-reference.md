# Language Reference

The language reference covers syntax and semantics that the compiler must understand
directly.

This area should answer questions such as:

- How is a construct parsed?
- What names or forms are legal in a declaration?
- How does evaluation proceed?
- What is a type error, parse error, or exhaustiveness error?
- Which parts are core semantics and which parts are merely library surfaces?
